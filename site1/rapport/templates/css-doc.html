Référence CSS

Utilisez cette référence CSS pour parcourir un index alphabétique de toutes les propriétés CSS standard, des pseudo-classes, des pseudo-éléments, des types de données, des notations fonctionnelles et des règles AT. Vous pouvez également parcourir les concepts CSS clés et une liste de sélecteurs organisés par type. Une brève référence DOM-CSS / CSSOM est également incluse.

Syntaxe de base des règles

Syntaxe des règles de style

style-rule ::=
    selectors-list {
      properties-list
    }

Où:

selectors-list ::=
    selector[:pseudo-class] [::pseudo-element]
    [, selectors-list]

properties-list ::=
    [property : value] [; properties-list]

Voir l’index des sélecteurs, pseudo-classes et pseudo-éléments ci-dessous. La syntaxe de chaque valeur spécifiée dépend du type de données défini pour chaque propriété spécifiée.

Exemples de règles de style

strong {
  color: red;
}

div.menu-bar li:hover > ul {
  display: block;
}

Pour une introduction de niveau débutant à la syntaxe des sélecteurs, consultez notre guide sur les sélecteurs CSS. N’oubliez pas que toute erreur de syntaxe dans une définition de règle invalide l’intégralité de la règle. Les règles non valides sont ignorées par le navigateur. Notez que les définitions de règles CSS sont entièrement basées sur du texte (ASCII), tandis que DOM-CSS / CSSOM (le système de gestion des règles) est basé sur des objets.

Syntaxe de la règle

Comme la structure des règles at-rules varie considérablement, veuillez consulter At-rule pour trouver la syntaxe de la règle spécifique que vous souhaitez.

Selectors

The following are the various selectors, which allow styles to be conditional based on various features of elements within the DOM.

Basic selectors

Basic selectors are fundamental selectors; these are the most basic selectors that are frequently combined to create other, more complex selectors.

  • Universal selector *
  • Type selector elementname
  • Class selector .classname
  • ID selector #idname
  • Attribute selector [attr=value]

Grouping selectors

Selector list A, B

Specifies that both and elements are selected. This is a grouping method to select several matching elements.AB

Combinators

Combinators are selectors that establish a relationship between two or more simple selectors, such as " is a child of " or " is adjacent to ", creating a complex selector.ABAB

Next-sibling combinator A + B

Specifies that the elements selected by both and have the same parent and that the element selected by immediately follows the element selected by horizontally.ABBA

Subsequent-sibling combinator A ~ B

Specifies that the elements selected by both and share the same parent and that the element selected by comes before—but not necessarily immediately before—the element selected by .ABAB

Child combinator A > B

Specifies that the element selected by is the direct child of the element selected by .BA

Descendant combinator A B

Specifies that the element selected by is a descendant of the element selected by , but is not necessarily a direct child.BA

Column combinator Experimental A || B

Specifies that the element selected by is located within the table column specified by . Elements which span multiple columns are considered to be a member of all of those columns.BA

Pseudo

Pseudo classes :

Specifies a special state of the selected element(s).

Pseudo elements ::

Represents entities that are not included in HTML.

See also selectors in the Selectors specification and the pseudo-element specification.
